Islands Architecture

HEADLINE

Pages are static HTML by default. Only interactive components ("islands") ship JavaScript, with fine-grained control over when they hydrate. Choose from 5 strategies: client:load, client:visible, client:idle, client:media, and client:none.

Dashboard (5 widgets) 120KB → ~8 KB

Islands Architecture

Pages are static HTML by default — zero JavaScript. Only interactive components ("islands") ship JS to the browser, with fine-grained control over when they hydrate.

✓ 0 KB JS on static pages — content visible instantly
✓ 5 hydration strategies — load, visible, idle, media, none
✓ Per-component JS — each island loads independently
✓ SSR-only islands — reusable components with zero JS
✓ ~1KB runtime — scans and hydrates automatically
Islands Docs →
pages/dashboard.html 
<!-- Static HTML — 0 KB JavaScript -->
<h1>Dashboard</h1>
<div class="stats">Revenue: ${{ .Props.stats.Revenue }}</div>

<!-- Loads immediately -->
{{ "{{" }} island "search" (props "endpoint" "/api/search") "client:load" {{ "}}" }}

<!-- Loads when scrolled into view -->
{{ "{{" }} island "chart" (props "data" .Props.chartData) "client:visible" {{ "}}" }}

<!-- Loads when browser is idle -->
{{ "{{" }} island "notifications" (props "count" .Props.unread) "client:idle" {{ "}}" }}

<!-- Only loads on mobile -->
{{ "{{" }} island "mobile-filters" nil "client:media=(max-width:768px)" {{ "}}" }}

<!-- SSR only — zero JS ever -->
{{ "{{" }} island "activity-feed" (props "events" .Props.events) "client:none" {{ "}}" }}
