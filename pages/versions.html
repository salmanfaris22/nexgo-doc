v2.2.0
April 2026

High-Performance Mode — 200k+ req/sec

Cluster mode, async logging, response caching, atomic template reads, and route match caching. Push throughput from 50k to 200k+ req/sec on multi-core machines. No breaking changes.

Performance

New
Cluster Mode

Multiple HTTP workers sharing one TCP listener. Auto-detects CPU cores. Zero-downtime restart.

New
Async Logging

Buffered channel (8,192 entries) with background writer. Non-blocking I/O on the hot path.

New
Response Caching

In-memory full response cache with TTL. X-Cache: HIT/MISS headers. Microsecond response times.

New
Route Match Caching

sync.Map-based cache skips regex matching for known paths. 15ns vs 800ns lookups.

New
Atomic Template Reads

Lock-free template reads via sync/atomic.Value. O(1) constant time regardless of concurrency.

New
HTTP Server Tuning

Configurable buffer sizes, timeouts, and keep-alive settings for high-throughput scenarios.

v2.1.0
April 2026

Islands Architecture — Zero JS by Default

Pages ship as pure static HTML. Only interactive components ("islands") load JavaScript, with 5 hydration strategies. Fully backwards compatible with v2.0.x — no breaking changes.

Islands Architecture

New
pkg/islands — Island Registry & Renderer

Full island registry with template rendering, JS serving, and hydration runtime.

New
5 Hydration Strategies

client:load, client:visible, client:idle, client:media, client:none — fine-grained control over when JS loads.

New
island, props, islandRuntime Template Functions

Render islands in templates with island "name" (props "k" "v") "client:visible".

New
Island JS Serving

/_nexgo/islands/{name}.js endpoint serves individual island JS files as ES modules.

New
~1KB Hydration Runtime

Lightweight runtime using IntersectionObserver, requestIdleCallback, and matchMedia for efficient hydration.

New
SSR-Only Islands

Create reusable components without a .js file — rendered server-side, zero JS shipped.

New
HMR for Islands

Hot reload support for island templates and JS files during development.

New
islandsDir Config

Configure the islands directory in nexgo.config.json. Defaults to "islands".

v1.2.0
April 2026

SEO, Streaming SSR, ISR, Worker Pools & Advanced Features

Production-ready features: SEO meta tags, sitemaps, streaming SSR, incremental static regeneration, worker pools, response caching, parallel data loading, and template caching.

New Features

New
SEO Package

Meta tags, OpenGraph, Twitter Cards, sitemap.xml, robots.txt, and JSON-LD structured data generation.

New
Streaming SSR

Send HTML progressively as it's ready — show the shell immediately, stream content later. Includes SSE support.

New
Incremental Static Regeneration (ISR)

Cache pages and rebuild them in the background after a configurable interval — like Next.js ISR.

New
Worker Pools

Run tasks concurrently with controlled parallelism. Map, ForEach, and global pool support.

New
Response Caching

Cache GET responses in memory with TTL. Automatic X-Cache: HIT/MISS headers.

New
Parallel Data Loading

Load multiple data sources concurrently and merge results — like Promise.all() in JS.

New
Template Caching

Templates compiled once and cached. Parallel template loading with worker pool.
