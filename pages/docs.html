Quick Start

Get a NexGo project running in under 2 minutes.

1. Install NexGo

${{ .Props.installCmd }}

⚠️ PATH Setup (Important):

export PATH=$PATH:$(go env GOPATH)/bin
echo 'export PATH=$PATH:$(go env GOPATH)/bin' >> ~/.bashrc && source ~/.bashrc

2. Create a New Project

nexgo create my-app
cd my-app
go mod tidy
nexgo dev

Running nexgo create my-app generates this structure:

my-app/
├── main.go                  # Application entry point
├── nexgo.config.json        # Framework configuration
├── go.mod
│
├── pages/                   # File-based routes
│   ├── index.html           #   → /
│   ├── about.html           #   → /about
│   ├── blog/
│   │   ├── index.html       #   → /blog
│   │   └── [slug].html      #   → /blog/:slug (dynamic)
│   └── api/
│       └── hello.go         #   → /api/hello
│
├── layouts/
│   └── default.html         # Default layout
├── components/              # Reusable partials

│   ├── css/global.css
│   └── js/
└── .nexgo/out/              # Build output (generated)

File-Based Routing

Drop HTML files in pages/ and NexGo creates routes automatically — no manual registration needed.

File Path URL Pattern Example
pages/index.html//
pages/about.html/about/about
pages/blog/index.html/blog/blog
pages/blog/[slug].html/blog/:slug/blog/my-post
pages/docs/[...path].html/docs/*/docs/api/auth
pages/api/users.go/api/users/api/users

Dynamic Routes

Use [param] syntax for dynamic URL segments:

pages/
├── users/
│   ├── [id].html          → /users/123
│   └── [id]/posts.html    → /users/123/posts
└── [...catchall].html     → /any/path/here

Access parameters in templates:

<h1>User {"{"}{ .Params.id }{"}"}</h1>
<p>Post: {"{"}{ .Params.slug }{"}"}</p>
Rendered Output — /users/42
localhost:3000/users/42

User 42

Post: my-first-blog

Template Engine

NexGo uses Go's html/template package. Every page receives a PageData struct:

type PageData struct {
    Title        string
    Description  string
    Path         string                 // Current URL path
    Params       map[string]string      // Route params
    Query        map[string]string      // Query string
    Props        map[string]interface{} // From DataLoader
    DevMode      bool
    BuildID      string
}

API Routes

Create .go files in pages/api/ to build REST endpoints:

    "net/http"
    "github.com/salmanfaris22/nexgo/pkg/api"

    router.RegisterAPI("/api/hello", Hello)
}

func Hello(w http.ResponseWriter, r *http.Request) {
    api.JSON(w, map[string]interface{}{
        "message": "Hello from NexGo!",
        "method":  r.Method,

Middleware

Built-in middleware is applied in this order:

Request → Recover → Logger → SecurityHeaders → [Gzip] → Handler
Server Log — Middleware in action
[NexGo] GET  /about       200  1.2ms
[NexGo] POST /api/users   201  3.8ms
[NexGo] GET  /not-found   404  0.4ms
[NexGo] GET  /api/crash   500  0.1ms recovered: nil pointer
MiddlewareDescription
RecoverCatches panics, returns 500
LoggerLogs method, path, status, duration
SecurityHeadersX-Content-Type-Options, X-Frame-Options, etc.
GzipCompresses responses (configurable)
CORS(origins...)Configurable CORS headers
Cache(maxAge)Sets Cache-Control headers

HTMX Helpers

Send HTML fragments from Go and let HTMX update the DOM — no page reloads needed.

api.IsHTMX(r) — Check if request is from HTMX

if api.IsHTMX(r) {
    // Return HTML fragment
    api.HTMXHTML(w, "<li>New item</li>")
} else {
    // Return JSON
    api.JSON(w, data)
}

api.HTMXHTML(w, html) — Send HTML fragment response

api.HTMXHTML(w, "<div>Updated content</div>")
HTTP Response — HTMX fragment
HTTP/1.1 200 OK
Content-Type: text/html

<div>Updated content</div>
↳ HTMX swaps this into the target element — no page reload

api.HTMXTrigger(w, event) — Fire client-side event after response

api.HTMXTrigger(w, "todo-added")

api.HTMXHeader(w, key, value) — Set any HTMX header

api.HTMXHeader(w, "HX-Redirect", "/dashboard")

Parallel Data Loading

Load multiple data sources concurrently and merge results — like Promise.all() in JS.

import (
    "github.com/salmanfaris22/nexgo/pkg/renderer"
    "net/http"
)

// Create parallel loader
loader := renderer.NewParallelLoader().
    Add("users", fetchUsers).
    Add("posts", fetchPosts).
    Add("stats", fetchStats)

// Execute all concurrently
props, err := loader.Execute(req, params)
// props = map[string]interface{}{
//     "users": [...], "posts": [...], "stats": {...}
// }

func fetchUsers(req *http.Request, params map[string]string) (map[string]interface{}, error) {
    users := db.Query("SELECT * FROM users")
    return map[string]interface{}{"users": users}, nil
}

func fetchPosts(req *http.Request, params map[string]string) (map[string]interface{}, error) {
    posts := db.Query("SELECT * FROM posts LIMIT 10")
    return map[string]interface{}{"posts": posts}, nil
}

Streaming SSR

Send HTML progressively as it's ready — show the shell immediately, stream content later.

import "github.com/salmanfaris22/nexgo/pkg/stream"

func handlePage(w http.ResponseWriter, r *http.Request) {
    sw := stream.StreamHTML(w)

    // Send head + layout immediately
    sw.WriteString(`<!DOCTYPE html><html><head><title>My Page</title></head><body>`)
    sw.WriteString(`<header>Navigation</header><main>`)
    sw.Flush()

    // Do slow work (DB query, API call)
    data := slowDatabaseQuery()

    // Stream content
    sw.WriteString(fmt.Sprintf("<h1>%s</h1>", data.Title))
    sw.WriteString(fmt.Sprintf("<p>%s</p>", data.Body))

    // Close HTML
    sw.WriteString(`</main></body></html>`)
}

Server-Sent Events (SSE)

func handleSSE(w http.ResponseWriter, r *http.Request) {
    sse := stream.NewSSE(w)
    defer sse.Flush()

    for {
        sse.Send("update", `{"count": 42}`)
        sse.Ping()
        time.Sleep(1 * time.Second)
    }
}

Response Caching

Cache GET responses in memory with TTL. Automatic cache headers.

Middleware

import "github.com/salmanfaris22/nexgo/pkg/api"

// Cache a handler for 5 minutes
mux.HandleFunc("/api/posts", api.Cache(5*time.Minute)(handlePosts))

// Cache with custom TTL
mux.HandleFunc("/api/slow", api.Cache(30*time.Second)(handleSlow))

Manual Cache

// Set cache
api.CacheSet("/api/data", 200, headers, body)

// Get cache
code, headers, body, ok := api.CacheGet("/api/data")
if ok {
    w.WriteHeader(code)
    w.Write(body)
    return
}

// Delete specific key
api.CacheDelete("/api/data")

// Clear all
api.CacheClear()
Response Headers — Cache status
First request:   X-Cache: MISS  — generated fresh (45ms)
Second request:  X-Cache: HIT   — served from cache (0.2ms)
After TTL:       X-Cache: MISS  — cache expired, regenerated

Worker Pools

Run tasks concurrently with controlled parallelism.

Basic Pool

import "github.com/salmanfaris22/nexgo/pkg/worker"

pool := worker.New(4)  // 4 workers
pool.Start()

pool.Submit(func() error {
    return processFile("data1.csv")
})
pool.Submit(func() error {
    return processFile("data2.csv")
})

errs := pool.Wait()  // blocks until all done

Map (concurrent transform)

results := worker.Map(4, items, func(item Item) Result {
    return process(item)
})
// results are in original order

Map with Errors

results, err := worker.MapErr(4, urls, func(url string) (Page, error) {
    return fetchPage(url)
})

ForEach

worker.ForEach(8, emails, func(email string) {
    sendEmail(email)
})

Global Pool

worker.Start()  // starts global pool with 4 workers
worker.Submit(func() error { return doWork() })
errs := worker.Wait()

Incremental Static Regeneration (ISR)

Cache pages and rebuild them in the background after a configurable interval — like Next.js ISR.

Basic ISR

import "github.com/salmanfaris22/nexgo/pkg/isr"

// Create ISR with 60-second revalidation
cache := isr.New(60 * time.Second)

func handlePage(w http.ResponseWriter, r *http.Request) {
    cache.Serve(w, r, func() (int, http.Header, []byte, error) {
        // This runs only on first request or after 60s
        html := renderExpensivePage()
        return 200, nil, []byte(html), nil
    })
}

How ISR Works

StepBehaviorHeader
First requestGenerates page, caches it, returnsX-ISR: MISS
Subsequent (within TTL)Serves cached pageX-ISR: STALE
After TTL expiresServes stale, regenerates in backgroundX-ISR: REVALIDATING
Next requestGets fresh cached contentX-ISR: HIT

Manual Revalidation

// Trigger background rebuild of a specific path
cache.Revalidate("/blog/post-1", func() (int, http.Header, []byte, error) {
    return 200, nil, []byte(renderPost()), nil
})

// Purge from cache
cache.Purge("/blog/post-1")
cache.PurgeAll()

Global ISR

isr.SetRevalidate(120 * time.Second)

isr.Serve(w, r, func() (int, http.Header, []byte, error) {
    return 200, nil, []byte(html), nil
})

isr.Revalidate("/page", generateFunc)
isr.Purge("/page")

Environment Variables

The pkg/env package loads .env files with priority-based overrides and provides type-safe access to environment variables. Files are loaded in order: .env, .env.local, .env.{mode}, with later files taking precedence.

import "github.com/salmanfaris22/nexgo/pkg/env"

// Load .env files (call once at startup)
env.LoadGlobal()              // loads .env, .env.local, .env.development
env.LoadGlobal("production")  // loads .env, .env.local, .env.production

// Create a store for typed access
store := env.NewStore()
store.Get("DATABASE_URL")              // returns value or ""
store.MustGet("SECRET_KEY")            // panics if missing
store.GetInt("PORT", 3000)             // with default
store.GetBool("DEBUG", false)
Example .env files & loaded values
# .env
DATABASE_URL=postgres://localhost/myapp
PORT=3000

# .env.production
DATABASE_URL=postgres://prod-db/myapp  ← overrides .env
SECRET_KEY=a1b2c3d4...

store.GetInt("PORT", 8080) → 3000
store.GetBool("DEBUG", false) → false

Sessions

Cookie-based session management with pluggable backends. Ships with MemoryStore (development) and FileStore (production). Supports flash messages for one-time notifications.

import "github.com/salmanfaris22/nexgo/pkg/session"

mgr := session.NewManager(session.Options{
    Store:    session.NewMemoryStore(),  // or session.NewFileStore("./sessions")
    MaxAge:   3600,
    HTTPOnly: true,
    Secure:   true,
})

func handler(w http.ResponseWriter, r *http.Request) {
    sess := mgr.Start(w, r)
    sess.Set("user", "alice")
    name := sess.Get("user")  // "alice"

    // Flash messages (auto-cleared after read)
    sess.Flash("success", "Item created!")
    msgs := sess.Flashes("success")  // ["Item created!"]
}

Authentication (JWT)

JWT token generation, verification, middleware, role-based access control, and password hashing via pkg/auth.

import "github.com/salmanfaris22/nexgo/pkg/auth"

// Generate a JWT
token, err := auth.GenerateToken(auth.Claims{
    UserID: "123",
    Role:   "admin",
}, "my-secret-key", 24*time.Hour)

// Verify a JWT
claims, err := auth.VerifyToken(token, "my-secret-key")

// Middleware: protect routes
mux.Handle("/dashboard", auth.RequireAuth("my-secret-key")(dashboardHandler))
mux.Handle("/admin", auth.RequireRole("admin", "my-secret-key")(adminHandler))

// Password hashing
hash, _ := auth.HashPassword("s3cret")
ok := auth.CheckPassword("s3cret", hash)  // true

CSRF Protection

Token-based CSRF protection via pkg/csrf. Generates a per-session token, validates it on POST/PUT/DELETE requests, and provides a template helper for forms.

import "github.com/salmanfaris22/nexgo/pkg/csrf"

// Apply middleware
handler := csrf.Protect("my-csrf-secret")(myHandler)

// In templates, use the helper to embed the token in forms:
// <form method="POST">
//   {"{"}{ csrfField }{"}"} <!-- renders hidden input -->
//   <button type="submit">Submit</button>
// </form>

// For AJAX, read the token from the meta tag or cookie:
// X-CSRF-Token: <token>

Rate Limiting

Protect endpoints from abuse with token bucket and sliding window algorithms via pkg/ratelimit. Apply as global middleware or per-route.

import "github.com/salmanfaris22/nexgo/pkg/ratelimit"

// Token bucket: 100 requests per minute per IP
limiter := ratelimit.PerMinute(100)
mux.Handle("/api/", limiter.Middleware()(apiHandler))

// Sliding window: 10 requests per second
strict := ratelimit.PerSecond(10)
mux.Handle("/api/login", strict.Middleware()(loginHandler))

// Custom configuration
limiter := ratelimit.New(ratelimit.Config{
    Rate:     50,
    Window:   time.Minute,
    Strategy: ratelimit.SlidingWindow,
    KeyFunc:  func(r *http.Request) string { return r.RemoteAddr },
})

Internationalization (i18n)

Locale routing, JSON translation files, RTL support, and automatic locale detection via pkg/i18n. Place translation files in locales/.

import "github.com/salmanfaris22/nexgo/pkg/i18n"

// locales/en.json: {"greeting": "Hello, {"{"}name{"}"}!", "nav.home": "Home"}
// locales/ar.json: {"greeting": "!{"{"}name{"}"} ,مرحبا", "nav.home": "الرئيسية"}

bundle := i18n.New(i18n.Config{
    DefaultLocale: "en",
    Locales:       []string{"en", "fr", "ar"},

// In handlers
t := bundle.T("en")
t.Get("greeting", map[string]string{"name": "World"}) // "Hello, World!"

// Locale detection middleware (Accept-Language, cookie, URL prefix)
mux.Handle("/", bundle.Middleware()(handler))

WebSocket

import "github.com/salmanfaris22/nexgo/pkg/websocket"

hub := websocket.NewHub()
go hub.Run()

mux.HandleFunc("/ws", func(w http.ResponseWriter, r *http.Request) {
    conn, err := websocket.Upgrade(w, r)
    if err != nil { return }
    client := hub.Register(conn)
    defer hub.Unregister(client)

    // Join a room
    hub.Join(client, "chat-room-1")

    // Broadcast to all clients
    hub.Broadcast([]byte("Hello everyone!"))

    // Broadcast to a specific room
    hub.BroadcastTo("chat-room-1", []byte("Room message"))
})

Creating Islands

An island is a pair of files in the islands/ directory:

FilePurposeRequired
islands/name.htmlServer-side Go templateYes

Step 1: Create the Template

<!-- islands/counter.html -->
<div class="counter">
  <span class="counter-value">{{ "{{" }} .count {{ "}}" }}</span>
  <button class="counter-btn" data-action="decrement">-</button>
  <button class="counter-btn" data-action="increment">+</button>

Step 2: Create the Client JS

// islands/counter.js
export default function init(el, props) {
  let count = props.count || 0;
  const display = el.querySelector('.counter-value');

  el.querySelectorAll('.counter-btn').forEach(btn => {
    btn.addEventListener('click', () => {
      count += btn.dataset.action === 'increment' ? 1 : -1;
      display.textContent = count;
    });
  });
}

Rules for island JS:

  • Export a default function or a named init function
  • Receives two arguments: el (the <nexgo-island> DOM element) and props
  • Use vanilla JS — no framework required (but you can import one)

Hydration Strategies

Hydration strategies control when an island's JavaScript loads and executes.

StrategyJS LoadedWhenBest For
client:loadImmediatelyPage loadAbove-the-fold interactive UI
client:visibleOn scrollEnter viewportBelow-the-fold widgets
client:idleWhen idleBrowser free timeNon-critical interactive UI
client:mediaOn matchMedia queryDevice-specific UI
client:noneNever—Reusable SSR-only components

client:load — Immediate

Hydrate immediately when the page loads. Use for above-the-fold interactive elements.

{{ "{{" }} island "search-bar" (props "placeholder" "Search...") "client:load" {{ "}}" }}

client:visible — On Scroll

Hydrate when the island scrolls into the viewport (using IntersectionObserver with 200px margin).

{{ "{{" }} island "comment-form" (props "postId" .Params.slug) "client:visible" {{ "}}" }}

client:idle — When Free

Hydrate when the browser is idle (using requestIdleCallback).

{{ "{{" }} island "analytics-widget" (props "data" .Props.stats) "client:idle" {{ "}}" }}

client:media — Device-Specific

Hydrate only when a CSS media query matches. The island JS is never loaded on non-matching devices.

{{ "{{" }} island "mobile-menu" nil "client:media=(max-width:768px)" {{ "}}" }}

client:none — SSR Only

Never hydrate. The island is rendered server-side and sent as static HTML. No JS is ever shipped.

{{ "{{" }} island "sidebar" (props "user" .State.user) "client:none" {{ "}}" }}

DOM Queries (Scoped to Island)

// islands/tabs.js
export default function(el, props) {
  const buttons = el.querySelectorAll('[data-tab]');
  const panels = el.querySelectorAll('[data-panel]');

  buttons.forEach(btn => {
    btn.addEventListener('click', () => {
      const target = btn.dataset.tab;
      panels.forEach(p => p.hidden = p.dataset.panel !== target);
      buttons.forEach(b => b.classList.toggle('active', b === btn));
    });
  });
}

<!-- islands/search.html -->

         placeholder="{{ "{{" }} .placeholder {{ "}}" }}" />
  <div class="search-results"></div>

// islands/search.js
export default function(el, props) {
  const input = el.querySelector('.search-input');
  const results = el.querySelector('.search-results');
  let timeout;

  input.addEventListener('input', () => {
    clearTimeout(timeout);
    timeout = setTimeout(async () => {
      const q = input.value.trim();
      const res = await fetch(`/api/search?q=${"{"}q{"}"}`);
      const data = await res.json();
      results.innerHTML = data.items
        .map(i => `<a href="${"{"}i.url{"}"}">${"{"}i.title{"}"}</a>`)
        .join('');
    }, 300);
  });
}

Island Runtime

The island runtime is a lightweight (~1KB) JavaScript file that handles discovery and hydration.

How It Works

  1. On page load, the runtime scans for <nexgo-island> elements
  2. For each island, it reads the data-strategy attribute
  3. Based on the strategy, it schedules hydration:
    • load → runs immediately
    • visible → sets up IntersectionObserver
    • idle → calls requestIdleCallback
    • media → listens to matchMedia
  4. When triggered, it dynamically imports the island's JS module
  5. The module's default export is called with (element, props)

Including the Runtime

<!-- Option A: Script tag (recommended) -->
<script src="/_nexgo/island-runtime.js" defer></script>

<!-- Option B: Inline via template function -->
{{ "{{" }} islandRuntime {{ "}}" }}

Migration from v2.0.x

No breaking changes. Islands are purely additive. Steps to start using them:

  1. Update NexGo: go get github.com/salmanfaris22/nexgo/v2@v2.1.0
  2. Create islands/ directory: mkdir islands
  3. Add the runtime to your layout: <script src="/_nexgo/island-runtime.js" defer></script>
  4. Start creating islands!
